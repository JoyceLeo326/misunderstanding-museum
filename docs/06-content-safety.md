# 06 · Content safety and privacy

**Status:** Eight content-safety principles · Fifteen-point release check

## Principle

> We can collect misunderstandings; we cannot consume other people’s vulnerability.

跨文化关系内容天然带有隐私、身份和刻板印象风险。情绪越强，越需要明确素材来源、编辑方式和当事人权利。

## Material labels

Every published item must use one of these labels:

- **Real, permissioned story** — 已完成授权、核对和匿名化。
- **Fictional composite** — 为展示方法而写作的虚构复合故事。
- **Editorial illustration** — 不指向任何真实人物的文化语境示例。
- **Creator partnership** — 清楚披露商业合作与品牌参与。

不允许用模糊的“用户故事”包装虚构案例。

## Privacy requirements

### Before collection

- 说明采集目的、公开范围、保存时间和删除方式。
- 只收集策展必需的信息，不默认要求真实姓名、学校、公司或位置。
- 聊天截图、照片、声音和关系故事分别获得授权。

### Before publication

- 删除姓名、账号、头像、二维码、位置、机构和敏感时间。
- 检查截图中第三人的信息。
- 将最终文案和视觉回传当事人确认。
- 明确二次分发、广告加热和跨平台使用是否包含在授权中。

### After publication

- 提供清晰的撤回与删除渠道。
- 限制原始素材访问权限。
- 撤回后停止新的二次分发，并按流程处理已有版本。

## Cultural sensitivity

- 描述“这一次发生了什么”，不写“这个国家的人都怎样”。
- 允许用户提供不同解释，不宣布一种关系语境是普遍真理。
- 不用口音、语言错误、国籍、种族或移民身份作为笑点。
- 不将跨国情侣矛盾娱乐化为“异国恋奇观”。
- K-pop与饭圈内容尊重社群用语，不用外部嘲讽视角获取流量。
- 涉及政治、宗教、身份和创伤时升级审核，不追求即时热点速度。

## AI and generated assets

- AI生成的人物、聊天或视觉要明确标注。
- 不用生成头像冒充真实用户。
- 不用合成截图伪装真实投稿记录或传播结果。
- 生成翻译示例必须由具备相应语言能力的人复核。
- 不把AI生成内容作为文化事实来源。

## Pre-publication checklist

### Source

- [ ] 内容是真实授权、虚构复合还是编辑示例？状态是否显著？
- [ ] 使用者是否有权提供所有素材？
- [ ] 最终版本是否由相关当事人确认？

### Privacy

- [ ] 姓名、账号、头像、位置、机构和敏感时间是否移除？
- [ ] 是否意外暴露第三人信息？
- [ ] 是否定义撤回、删除和二次分发范围？

### Context

- [ ] 字面、人物理解与编辑注释是否清楚分开？
- [ ] 语境解释是否来自当事人或可核验来源？
- [ ] 是否允许不同地区与关系身份提供其他解释？

### Culture

- [ ] 这是在解释差异，还是在制造刻板印象？
- [ ] 是否把个体经历概括成民族性格？
- [ ] 当事人是否可能因发布受到现实伤害？

### Data

- [ ] 阅读、互动、投稿与合作数据是否来自对应平台或台账？
- [ ] 指标是否注明样本、时间、预算、归因窗和口径？
- [ ] “待验证”内容是否被误写成“已证明”？

任何一项无法确认时，不发布或升级给负责人审核。
